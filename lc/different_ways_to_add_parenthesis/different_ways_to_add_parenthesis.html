Solutions are biased by problems

‚Äì Me

Problem Statement

Given a string expression of numbers and operators, return all possible results from computing all the different possible ways to group numbers and operators. You may return the answer in any order.

Example 1:

Input: expression = "2-1-1"
Output: [0,2]
Explanation:
((2-1)-1) = 0 
(2-(1-1)) = 2

Example 2:

Input: expression = "2*3-4*5"
Output: [-34,-14,-10,-10,10]
Explanation:
(2*(3-(4*5))) = -34 
((2*3)-(4*5)) = -14 
((2*(3-4))*5) = -10 
(2*((3-4)*5)) = -10 
(((2*3)-4)*5) = 10

go_figure {% github_emoji thinking %}

I got confused and took a wrong path here. I call it question bias

Initial thoughts on the problem statement:

  1. The question framed me to create a bracketted string before solving it.
  2. I saw a sub problem to the right of every position. If I solve it, I can bubble up and solve the given problem(It could be dp).
  3. I need a function to solve bracketed strings (this will be a stack solution)

I was working twice as hard (creating bracketted string, solving bracketted string). üêü

‚úîÔ∏è Instead of creating bracketed strings, I can directly start calculating for the solution. Fortunately this thought also helped me find an error in above proposed solution.

Eventually got around the correct thought process,

  1. Sub problem thinking was the issue. Though it has subproblems, but not only to right but also to left for every position
  2. So, for each non integer place, we can recursively solve for left and right and combine both solution (many to many operations)

var diffWaysToCompute = function(expression) {

    const ops = {
        "+": (l,r) => l + r,
        "-": (l,r) => l - r,
        "*": (l,r) => l * r
    }
    const solMap = {}
    const gen = (str) => {        
        if (solMap[str]) {
            return solMap[str]
        }
        if (!str.includes("*") && !str.includes("+") && !str.includes("-")) {
            solMap[str] = [parseInt(str) || 0]
            return [parseInt(str) || 0]
        }
        let sols = [];
        for (let i = 0; i < str.length; i++) {            
            if (isNaN(str[i])) {
                let right = gen(str.slice(i+1))
                let left = gen(str.slice(0,i))
                for (let l of left) {
                    for (let r of right) {
                        sols.push(ops[str[i]](l, r))    
                    }
                }


            }
        }

        solMap[str] = sols
        return sols
    }

    const ways = gen(expression)
    return ways
};
